================================================================================
                        HOW IT WORKS
================================================================================

User enters password
    ↓
Frontend checks:
  - Length >= 8 characters
  - Byte length <= 72 bytes
    ↓
If valid → Send to backend
If invalid → Show error
    ↓
Backend receives password
    ↓
Backend validates:
  - Pydantic checks (min 8, max 72)
  - Custom validator checks bytes
    ↓
If valid → Hash with bcrypt
If invalid → Return error

================================================================================
                        IMPORTANT: BYTES vs CHARACTERS
================================================================================

Characters = What you see
  "hello" = 5 characters

Bytes = How it's stored
  "hello" = 5 bytes (ASCII)
  "你好" = 6 bytes (Chinese, 2 characters)
  "😀" = 4 bytes (emoji, 1 character)

bcrypt limit = 72 BYTES (not characters)

Examples:
  password123 = 11 bytes ✅
  verylongpassword = 16 bytes ✅
  你好世界你好世界... (36 chars) = 108 bytes ❌ (too long)
  😀😀😀😀😀😀😀😀😀😀😀😀😀😀😀😀😀😀 = 72 bytes ✅ (exactly at limit)
